Features - Error Detection/Correction after Input
------------------------------------------------------
1) Detection of items that have only purchased once (or perhaps less than X amount of times). 
   Since we expect most items to be purchases often, it an items has few occurences it might represent an error from the OCR.
2) Similar description (or item code) search algorithm (triggered manually or automatically). If entries have similar enough descriptions or items codes 
   they might actually be the same item and it was just on OCR error.
3) Price anonamilies. Add a feature to check if the price of an item went up for a single instance and then come back down to what it previously was.
   If this is the case, it might represent an OCR error causing an the anomaly.
4) Put commonly seen OCR errors (for example, confusing '5' with '6') into a database. When price anonamilies, or similar descriptions or items codes, are detected check if any of the commonly seen OCR 
   errors could be present and have caused the error. Given this information, we might be able to auto correct the error, or at least suggest a correction to the user.


Features - Error Detection/Correction at Input
-----------------------------------------------
* While the the input verification that is below should reduce errors, it is still possible to have the wrong values add to a correct sum, so the error detection/correction after input is still needed.
  If errors get past the input verification, hopefully the detection/correction strategy after input can slowly catch the missed errors and cleanup the dataset over time. *
1) Get user to input the total price (before tax) when inputting a photo. Then when the software processes the photo, have it sum all of the items it found 
   and compare against the total that was inputted by the user. If these values are different, an error is present.
2) If an error is detected (from (1) above), use automatic detection/correction techniques similar to those from "Features - Error Detection/Correction after Input"
  - Compare each item code and description to item codes and description already present in the database. Correct ones that are similar but slighlty wrong to the matching 
  previously seen one in the database. If multiple similar ones are found, prompt the user to correct manually.
  - If a price anomaly is detected, check commonly known OCR errors and see if appling an automatic fix removes the price anomaly. If the price anomaly can't be fixed automatically
  prompt the user to enter the correction
3) If an error is detected by the total price check, after the auto and manual corrections are fixed, rerun the price check with the corrected list of items. Rerun the error detection/correction cycle until 
   the sum of the items matches the user inputted pre-tax total.
